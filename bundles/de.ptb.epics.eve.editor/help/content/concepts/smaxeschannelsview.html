The content of the SM Axes / Channels View depends on the type of the currently selected scan module. It shows attributes that can be altered and allows adding/deleting devices, actions, events, etc. in certain cases. The different appearances are shown in the following.

Scan Module

back to top

For a common (a.k.a. classic) scan module the SM Axes / Channels View looks as follows:

General Settings

  • No of Measurements: the number of measurements taken for each motor position
  • Trigger Delay (in s): delay in s before detectors are triggered
  • Settle Time (in s): delay in s after the first motor position is reached
  • Manual Trigger: whether setting motor positions and/or triggering detectors should be confirmed

Motor Axes

Allows adding/removing of and displays Motor Axes of the currently selected scan module. Properties of selected axes can be modified either in the Motor Axis View or directly in the table.

Detector Channels

Allows adding/removing of and displays Detector Channels of the currently selected scan module. Properties of selected channels can be modified either in the Detector Channel View or directly in the table.

Static Axis Snapshot

back to top

When a scan module of type “Axis Snapshot (static)” is selected, the view shows a table containing all axes contained in the snapshot. Axes can be added or removed.

Static Channel Snapshot

back to top

When a scan module of type “Channel Snapshot (static)” is selected the view shows a table containing all channels contained in the snapshot. Channels can be added or removed.

Dynamic Axis Snapshot

back to top

Scan modules of type “Axis Snapshot (dynamic)” cannot be altered. Their contents is determined during scan execution. The view shows an immutable table showing all axes contained in the snapshot based on the currently loaded device definition.

Dynamic Channel Snapshot

back to top

Scan modules of type “Channel Snapshot (dynamic)” cannot be altered. Their contents is determined during scan execution. The view shows an immutable table showing all channels contained in the snapshot based on the currently loaded device definition.
