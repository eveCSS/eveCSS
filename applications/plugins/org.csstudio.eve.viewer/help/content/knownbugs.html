(#2430) DeviceOptionsView doesn’t show selected device’s options

If a DeviceOptionsView is opened via the Window -> Show View Menu it doesn’t work properly. This is due to Eclipse Bug #294066: [Commands] org.eclipse.ui.views.showView command should have a secondaryId parameter.

Workaround: Open DeviceOptionViews via the ContextMenu of a Device in the Devices or DeviceInspector Views.

(#1471) PV channel transport closed when using DBE_PROPERTY monitor

When DBE_PROPERTY monitoring is set in CSS Core -> Epics preferences and a CA gateway with EPICS base 3.14.12.2 is used, PV access won’t work properly. For more information see Using CAJ in production (DBE_PROPERTY and CA gateway) and bad resource id msg from portable cas

(#523) No file name in Engine View under certain circumstances

If the following conditions are met, no file name is given in the engine view for the current scan:

  • an engine process is running
  • eveCSS is started showing the EVE Editor perspective
  • a scan is send to the engine via the „Add to playlist and switch to Engine Perspective” button

Workaround: Disconnect and Reconnect the engine via the respective buttons.

(#244) Device Inspector opened by Window -> Show View -> Other... -> EVE Viewer -> Device Inspector does not accept devices added per double click.

If all previous Device Inspector Views were closed and a new one is added via the above mentioned Window menu it won’t accept device by double clicking them in the Devices View.

Workaround: Open a Device Inspector by the above mentioned method. Then add another one with the toolbar button. Close the first one. Use the second one.
