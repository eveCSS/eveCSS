Device Description

A device description (file) is a collection of devices defined in XML. It is a list of motors and axes, detectors and channels, devices, options, plug ins and events. A device description file is required in order to run the EVE application.

Motors & Motor Axes

Motors are devices where you can read from or write to (a ‚real’ motor would actually move its axis or a radiator alter its temperature). They contain at least one axis.

Detectors & Detector Channels

Detectors are devices to measure something. They have at least one channel. During the main phase they can only be read. Additionally options can be set during pre and postscan phases.

(Pre-/Postscan) Device

Pre-/Postscan devices are devices where values could be assigned (as the name suspects) exclusively in pre- and postscan phases (e.g. open/close a beam shutter or switch on/off a fan).

Option

Motors & motor axes as well as detectors & detector channels can have several options (e.g. GoTo Position or Hi Warn Limit). They can be assigned in pre- and postscan phases.

Plugin

There are several types of plug ins which allow extension of functionality. Examples are save plug ins (which allow saving measurement data in different formats, e.g. ASCII or HDF) and position plug ins (which define the way devices are positioned, e.g. GOTO-Peak).

Event

An event can start a scan (special S-0 event), pause, cancel a chain or scan module or repeat the last step. Depending on their use, they are assigned to a chain, a scan module or to a detector. Read more on the usage of events.
